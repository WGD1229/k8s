设置固定IP

虚拟机IP如果不设置，每次关闭又开启，有时候会重新分配IP，防止这种情况发生，首先固定IP地址

进入虚拟机，登录root（后面方便操作，也可以给/etc/sysconfig添加用户权限，用sudo操作），执行命令：

x
vim  /etc/sysconfig/network-scripts/ifcfg-ens33

按如下编辑：

xxxxxxxxxx
TYPE="Ethernet"
PROXY_METHOD="none"
BROWSER_ONLY="no"
BOOTPROTO="static"         # 使用静态IP地址，默认为dhcp
IPADDR="192.168.184.158"   # 设置的静态IP地址 我的是"192.168.184.158"
NETMASK="255.255.255.0"    # 子网掩码
GATEWAY="192.168.184.2"    # 网关地址 命令 netstat -rn 查看
DNS1="192.168.184.2"       # DNS服务器
DEFROUTE="yes"
IPV4_FAILURE_FATAL="no"
IPV6INIT="yes"
IPV6_AUTOCONF="yes"
IPV6_DEFROUTE="yes"
IPV6_FAILURE_FATAL="no"
IPV6_ADDR_GEN_MODE="stable-privacy"
NAME="ens33"
UUID="95b614cd-79b0-4755-b08d-99f1cca7271b"
DEVICE="ens33"
ONBOOT="yes"             #是否开机启用

保存后，重启network 

xxxxxxxxxx
systemctl restart network

用 ip a 命令查看自己的IP地址，此时IP地址会变为自己更改的IP，如果有两个IP，下面有lft xxx sec 描述，说明是临时IP，过一段时间会自己消失。

Kubernetes安装过程中一个很大的问题，相关组件的镜像都是托管在Google Container Registry上的。国内的镜像加速一般针对的是Dockerhub上的镜像。所以国内的服务器是没法直接安装GCR上的镜像的。 注意，这里即使翻墙之后，在执行初始化时仍然会报错

最后的解决办法是找到了一个在hangzhou.aliyuncs.com上的一个仓库（无语了），在上面下载相关的镜像之后，通过docker tag 之后就可以初始化kubernetes。

其实最好的还是你所在的公司如果用K8S，应该有相应的私有仓库，里面有相关的images可以直接用！

使用kubeadm init初始化集群 (只在主节点执行)

初始化前确认 kubelet启动和 cgroup-driver等方式是否对应。

接下来使用kubeadm初始化集群，选择k8s作为Master Node 确保没有设置http_proxy和https_proxy代理 在k8s上执行下面的命令：

xxxxxxxxxx
kubeadm init   --kubernetes-version=v1.15.1   --pod-network-cidr=10.244.0.0/16   --apiserver-advertise-address=192.168.184.158 --token-ttl 0

—kubernetes-version根据上面安装成功时的提示:kubectl.x86_64 0:1.11.2-0对应版本。 因为我们选择flannel作为Pod网络插件，所以上面的命令指定–pod-network-cidr=10.244.0.0/16。 对于某些网络解决方案，Kubernetes master 也可以为每个节点分配网络范围（CIDR），这包括一些云提供商和 flannel。通过 —pod-network-cidr 参数指定的子网范围将被分解并发送给每个 node。这个范围应该使用最小的 /16，以让 controller-manager 能够给集群中的每个 node 分配 /24 的子网。如果我们是通过 这个 manifest 文件来使用 flannel，那么您应该使用 —pod-network-cidr=10.244.0.0/16。大部分基于 CNI 的网络解决方案都不需要这个参数。 —apiserver-advertise-address是apiserver的通信地址，一般使用master的ip地址。

通过kubeadm初始化后，都会提供节点加入k8s集群的token。默认token的有效期为24小时，当过期之后，该token就不可用了，需要重新生成token，会比较麻烦，这里—token-ttl设置为0表示永不过期。

可能遇到的问题-卡在preflight/images

init过程中kubernetes会自动从google服务器中下载相关的docker镜像 kubeadm init过程首先会检查代理服务器，确定跟 kube-apiserver 的 https 连接方式，如果有代理设置，会提出警告。 如果一直卡在preflight/images

配置数据包转发

出于安全考虑，Linux系统默认是禁止数据包转发的。所谓转发即当主机拥有多于一块的网卡时，其中一块收到数据包，根据数据包的目的ip地址将包发往本机另一网卡，该网卡根据路由表继续发送数据包。这通常就是路由器所要实现的功能。

echo "1" > /proc/sys/net/ipv4/ip_forward

然后在/etc/sysconfig/network脚本中添加 FORWARD_IPV4="YES"

安装flannel network add-on：

x
docker pull quay.io/coreos/flannel:v0.10.0-amd64 
mkdir -p /etc/cni/net.d/
cat <<EOF> /etc/cni/net.d/10-flannel.conf
{"name":"cbr0","type":"flannel","delegate": {"isDefaultGateway": true}}
EOF
​
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/v0.9.1/Documentation/kube-flannel.yml

这里可以cat查看kube-flannel.yml这个文件里的flannel的镜像的版本

cat kube-flannel.yml|grep quay.io/coreos/flannel

如果Node有多个网卡的话，目前需要在kube-flannel.yml中使用—iface参数指定集群主机内网网卡的名称，否则可能会出现dns无法解析。需要将kube-flannel.yml下载到本地，flanneld启动参数加上—iface=

containers:
      - name: kube-flannel
        image: quay.io/coreos/flannel:v0.10.0-amd64
        command:
        - /opt/bin/flanneld
        args:
        - --ip-masq
        - --kube-subnet-mgr
        - --iface=eth1
